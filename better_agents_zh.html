Better Agents

Better Agents 是一個 CLI 工具，也是一套用於建構 AI 代理（Agent）的標準。

它能為您的程式碼助手（如 Kilocode, Claude Code, Cursor 等）增強能力，使其成為您選擇的任何代理框架（如 Agno, Mastra 等）及其最佳實踐的專家。

這是開始任何新代理專案的最佳方式。

Better Agent 架構

Better Agent 的結構和生成的 AGENTS.md 確保了業界的最佳實踐，讓您的代理準備好投入生產環境：

  • Scenario：為每個功能編寫代理測試，確保代理行為符合預期。
  • 提示詞版本控制：方便團隊協作。
  • 評估筆記本 (Evaluation notebooks)：用於測量特定提示詞的效能。
  • 完整的可觀測性：已預先配置好儀器。
  • 標準化結構：提高專案的可維護性。

專案結構範例

my-agent-project/
├── app/ (或 src/)      # 實際的代理程式碼，依據選擇的框架而定
├── tests/
│   ├── evaluations/    # 用於評估的 Jupyter notebooks
│   │   └── example_eval.ipynb
│   └── scenarios/      # 端對端情境測試
│       └── example_scenario.test.{py,ts}
├── prompts/            # 用於團隊協作的版本化提示詞檔案
│   └── sample_prompt.yaml
├── prompts.json        # 提示詞註冊表
├── .mcp.json           # MCP 伺服器配置
├── AGENTS.md           # 開發指南
├── .env                # 環境變數
└── .gitignore

AGENTS.md 中的結構和指南確保程式碼助手所需的每個新功能都經過適當的測試、評估，並且提示詞受到版本控制。

.mcp.json 附帶了所有正確的 MCP 設置，因此您的程式碼助手將成為您選擇的框架的專家，並知道在哪裡可以找到新工具。

快速開始 (Getting Started)

安裝

npm install -g @langwatch/better-agents

或者使用 npx：

npx @langwatch/better-agents init my-agent-project

初始化新專案

# 在當前目錄初始化
better-agents init .

# 在新目錄初始化
better-agents init my-awesome-agent

CLI 將引導您選擇程式語言、代理框架、程式碼助手、LLM 提供商和 API 金鑰。

需求 (Requirements)

注意 (Windows 使用者)：

由於此工具依賴 posix 套件，目前無法直接在 Windows 環境下安裝（會出現 unistd.h 缺失錯誤）。

解決方案：請使用 WSL (Windows Subsystem for Linux)。

⚠️ 關鍵檢查：即使在 WSL 中，也可能誤用到 Windows 的 Node.js。請執行以下指令檢查：

which node

如果輸出的路徑包含 /mnt/c/...，代表您正在使用 Windows 版本的 Node.js，這會導致失敗。

正確做法：您必須在 WSL 內部安裝 Linux 版本的 Node.js (例如使用 nvm)。正確的路徑應該類似 /home/username/.nvm/... 或 /usr/bin/node。

  • Node.js 22+
  • npm 或 pnpm
  • 程式碼助手（以下擇一）：
    • Claude Code (claude CLI)
    • Cursor
    • Kilocode CLI (kilocode)
  • API 金鑰：
    • 您選擇的 LLM 提供商 API 金鑰
    • LangWatch API 金鑰 (在 這裡 獲取)
    • Smithery API 金鑰 (選填 - 用於 MCP 工具自動發現，在 這裡 獲取)
